# v0 By Vercel

## UI Role

v0 is the gold standard for generative UI layout. It combines conversational intent with immediate generated React output.

## AI Layout Pattern

- Left pane contains the chat and prompt iteration loop.
- Right pane contains the generated component, preview, and code-oriented artifact.
- The chat is visually secondary to the output.
- The generated UI is treated as the product surface, not just a message response.

## Why It Works

- The split-pane structure keeps cause and effect visible: prompt on one side, result on the other.
- Expanding and collapsing panels keep focus on the generated output.
- Minimal monochromatic borders prevent the assistant UI from competing with the preview.
- The product feels like a workspace, not a chatbot.

## Component Polish Notes

- Use thin separators rather than heavy containers.
- Keep controls visually quiet around the generated preview.
- Use compact toolbar treatments near the artifact, not large card headers.
- Make the output panel feel like a canvas.
- State changes should be obvious but not loud: streaming, pending, generated, edited, copied.

## Skill-Building Takeaway

For AI UI generation, design the system around first-class artifacts: previews, code files, versions, and history. The chat is the input surface; the artifact is the main interface.

## Detailed UI Construction Notes

v0 should be studied as an artifact-first split workspace. The left side should feel like an input rail, not a full app. A practical recreation would keep the prompt/chat column between 320px and 420px on desktop, with the artifact area taking every remaining pixel. The seam between panes should be a 1px low-contrast border, not a thick divider. The right pane should visually dominate by size, but not by louder styling.

The chat composer should sit close to the lower edge, with enough padding to feel stable: roughly 16px to 24px around the composer, 10px to 14px vertical padding inside the input, and a border radius around 12px to 16px. Message bubbles should be restrained. Avoid large colorful assistant bubbles; use text blocks, subtle backgrounds, and lightweight spacing instead.

The artifact panel should use a canvas treatment. Toolbars should be short, often 36px to 44px high, with small segmented controls for preview/code/props. Icons should be 14px to 16px, labels 12px to 13px. The output preview area should avoid extra nested cards. The component itself is the visual subject, so surrounding UI must be nearly invisible.

Component chrome should use monochrome borders: `border-subtle`, `text-muted`, and `surface-elevated` tokens. Avoid primary-color decoration except for active actions and generation states. A selected tab can use a barely raised surface plus stronger text rather than a bright underline.

Loading and streaming should feel procedural. Use skeleton lines inside the chat rail and a soft pending state in the preview area. Avoid spinners in the center of the canvas unless there is no partial output. If code is streaming, the preview should either show a stable placeholder or transition only once the artifact can render without layout thrash.

What to copy: split ownership between prompt and artifact, quiet borders, compact preview toolbar, generated output as first-class content.

What to avoid: making the chat rail visually equal to the preview, placing too many global actions around the artifact, using colorful assistant branding, or boxing the generated component inside multiple containers.
